# Instructions

1. **Shared Types (`src/shared/types/settings.ts`):**
   - Define `AppSettings` interface (`defaultTemplate: string`, `defaultPath: string`).
   - Define `DEFAULT_SETTINGS: AppSettings = { defaultTemplate: 'electron-forge-vite', defaultPath: '~/RisoTronProjects' }`.

2. **Config Service (`src/main/services/ConfigService.ts`):**
   - Create `ConfigService` that takes `DatabasePort` in its constructor.
   - Implement `safeJsonStringify` and `safeJsonParse` to reject `undefined`, `BigInt`, `NaN`, `Infinity` to prevent DB crashes.
   - `setSetting<K extends keyof AppSettings>(key: K, value: AppSettings[K])`: Uses UPSERT to ensure `updated_at` fires:
     `INSERT INTO settings (key, value, updated_at) VALUES (?, ?, CURRENT_TIMESTAMP) ON CONFLICT(key) DO UPDATE SET value = excluded.value, updated_at = CURRENT_TIMESTAMP`
   - `getAllSettings(): AppSettings`: Returns `{ ...DEFAULT_SETTINGS, ...dbRows }`.
   - `getSetting<K extends keyof AppSettings>(key: K): AppSettings[K]`.

3. **IPC Integration (`src/main/index.ts` & `src/preload/index.ts`):**
   - **Main:** Initialize `ConfigService`. Register `app:get-settings`, `app:get-setting`, `app:set-setting`.
   - **Main:** Add `validateSender(event)` to all new IPC handlers. Validate the `key` argument against `keyof AppSettings`.
   - **Preload:** Merge `settings` object into the existing `contextBridge.exposeInMainWorld('api', {...})`. Do NOT add a second `exposeInMainWorld` call. Expose `getAll()`, `get(key)`, `set(key, value)`.

4. **VSCode-Style UI Foundation (`src/renderer/styles/vscode-theme.css`):**
   - Create CSS variables matching VSCode dark theme:
     `--vscode-activity-bar: #333333; --vscode-sidebar: #252526; --vscode-editor: #1e1e1e; --vscode-status-bar: #007acc; --vscode-fg: #cccccc; --vscode-active: #ffffff;`
   - Import this in `src/renderer/App.svelte` or `src/renderer/main.ts`.

5. **Settings View (`src/renderer/components/SettingsView.svelte`):**
   - Build a settings page that uses the VSCode CSS variables.
   - Fetch settings on mount via `window.api.settings.getAll()`.
   - Create input fields for `defaultTemplate` and `defaultPath`.
   - Save via `window.api.settings.set(key, value)` when inputs change.
   - For now, you can just render `SettingsView` directly in `App.svelte` to test it.
